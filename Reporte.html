Introduccin

La Receita Federal do Brasil,la autoridad fiscal de Brasil publica la informacin del Cadastro Nacional da Pessoa Jurdica (CNP J - Registro Nacional de Personas Jurdicas) desde junio de 2023. Esta informacin es de gran utilidad para las empresas, ya que les ayuda a evaluar de manera imparcial su contexto, a sus socios, cuestiones legales y fiscales, posibles impactos de caso de alguna asociacin, etc. Este alojamiento de informacin ha implicado no solo grandes retos por parte del gobierno Brasileo, si no tambin a aquellos que quieren analizar su infomacin ya que esta, al da de hoy abarca aproximadamente 60 millones de empresas y mes con mes se agrega ms informacin. Debido a esto, las nuevas tecnologas y herramientas son primordiales para el almacenamiento y consulta de informacin, ya que medios tradicionales computaciones no consiguen ser lo bastante potentes para explotar el conocimiento que se aloja en los servidores de La Receita Federal do Brasil,la autoridad fiscal de Brasil.

Debido a esta problematica, en este trabajo se muestra el uso de la metodologa ETL, Extract, Transform and Loan, la cual es de vital importancia en el matenimiento y manejo de informacin en sistemas importantes de datos. Llevar esta metodologa implica retos computaciones, ya que se deben tener diferentes habilidades y experiencia para crear e implementar pipelines robustos que se ajusten a los requerimientos de los datos. En este caso, solo se toman los datos del mes de abril del ao 2025, los cuales abarcan ms de 20 GB de informacin.

La fuente de datos de este proyecto es el siguiente enlace y en este sitio se pueden encontrar direcciones de descarga a cada uno de los conjuntos de datos, comprimidos, que se requieren. Estos contienen informacin, segregada, de CNJP en la siguiente estructura:

def unzip_file(zip_file: str, input_dir: str, output_dir: str, new_name: str) -> None:
    """Extraccin y renombramiento de archivos comprimidos con un nombre indicado.

    Args:
        zip_file (str): Archivo a descomprimir.
        input_dir (str): Directorio en donde se ubican los archivos comprimidos.
        output_dir (str): Directorio en donde se guardarn los archivos con extensin csv.
        new_name (str): Nombre con el que se guardar el archivo.
    """
    # Por temas de identificacin del archivo extraido, se guarda en un folder temporal
    temp_dir = "temp_dir/"
    # Descrompresin en la carpeta temporal
    subprocess.run(["unzip", "-q", os.path.join(input_dir, zip_file), "-d", temp_dir], check=True)
    # Solo hay un archivo en dicho folder, el extrado anteriormente
    extracted_file = os.listdir("temp_dir/")[0]  
    # Cambiamos la extensin a csv y lo movemos a la carpeta destino
    os.rename(os.path.join(temp_dir, extracted_file), os.path.join(output_dir, new_name))
    # Se elimina el directorio temporal
    os.rmdir(temp_dir)

def read_large_file(file_path: str, chunk_size: int, colnames: list):
    """Generador para cargar por lotes un archivo csv

    Args:
        file_path (str): Ruta del archivo
        chunk_size (int): Nmero de registros a considerar por lote
        colnames (list): Nombres de las columnas

    Yields:
        pd.DataFrame: DataFrame con n (chunk_size) registros
    """
    for chunk in pd.read_csv(file_path, chunksize=chunk_size, encoding='latin-1',  sep=';', names=colnames, header=None, dtype=str):
        yield chunk

def sampling_a_big_file(name: str, col_names: list, input_dir: str, size_sample: int = 50, chunk_size: int = 10000) -> pd.DataFrame:
    """Funcin de muestro en un solo archivo de tamao considerable.

    Args:
        name (str): Nombre del archivo.
        col_names (list): Nombres de las columnas
        input_dir (str):  Directorio en donde se ubica el archivo 
        size_sample (int, optional): Tamao de la muestra. Defaults to 50.
        chunk_size (int, optional): Nmero de registros a considerar por lote. Defaults to 10000.

    Returns:
        pd.DataFrame: Muestra de el archivo de ingesta
    """
    # En resumen, para cada segmento de el archivo de ingesta, se obtiene una muestra y se va uniendo iterativamente las muestras
    # en un solo data frame. 
    return(
        reduce(
            lambda data1, data2: pd.concat([data1, data2], ignore_index=True, axis=0), 
            map(lambda batch_data: batch_data.sample(n=size_sample, random_state=9), read_large_file(os.path.join(input_dir, name), chunk_size, col_names)))
        )

def count_lines_vars(path: str, header: bool = False) -> int:
    """Funcin para contar filas de un archivo sin necesidad de cargarlo a memoria

    Args:
        path (str): Ruta del archivo
        header (bool, optional): Indicador de encabezado. Defaults to False.

    Returns:
        int: Nmero de registros en el archivo
    """
    # Con lo siguiente se cuenta el nmero de renglones 
    with open(path, "r", encoding="latin-1") as f:
        # Bsicamente, se recorre cada rengln y por cada rengln se va registrando un 1 que se va sumando iterativamente
        total = sum(1 for _ in f)
    n_rows = total - 1 if header else total
    # Con lo siguiente se cuenta el nmero de columnas
    count_columns = set()
    with open(path, encoding="latin-1") as f:
        reader = csv.reader(f, delimiter=';')
        for row in reader:
            count_columns.add(len(row))
    # Se devuelve el nmero de renglones y de columnas
    return n_rows, count_columns

def fix_date(date: str):
    """Funcin corregir fechas y llevar las que tengan formato 00000000 a nulos
    Args:
        date (str): Fecha  
    """
    if date == 00000000 or pd.isna(date):
        return pd.NaT
    try:
        return pd.to_datetime(date, format="%Y%m%d", errors="coerce")
    except:
        return pd.NaT
